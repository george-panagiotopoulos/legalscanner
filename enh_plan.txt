### 1. SBOM/SPDX Document Export
**Priority: CRITICAL ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê**

#### Why This Matters
- **Industry Standard**: SPDX is the Linux Foundation standard for software bill of materials
- **Compliance Requirement**: Many organizations REQUIRE SPDX/SBOM for legal approval
- **Ecosystem Integration**: SPDX files integrate with vulnerability scanners, dependency analyzers, and compliance platforms (GitHub Dependency Graph, Snyk, Grype, etc.)
- **Customer Value**: You already collect all the data‚Äîyou're just not exporting it in the standard format

#### Benefits
‚úÖ **Immediate compliance value** - makes your tool enterprise-ready
‚úÖ **Tool interoperability** - integrates with existing security/compliance workflows
‚úÖ **Legal team adoption** - lawyers/compliance officers know SPDX
‚úÖ **Differentiator** - not all scanners export SPDX properly

#### Open Source Library Recommendation
**Primary: `spdx` crate (Rust)**
- Crate: https://crates.io/crates/spdx
- Pure Rust SPDX 2.3 library
- Supports JSON, YAML, Tag-Value formats
- Type-safe builders for documents
- Maturity: ‚≠ê‚≠ê‚≠ê Good
- Ease of Use: ‚≠ê‚≠ê‚≠ê‚≠ê Easy

**Alternative: Custom serializer**
- SPDX JSON schema is well-documented
- Full control over output format
- ~500 lines of code to implement properly
- Good if you need specific customizations

#### Implementation Estimate
- **Complexity**: Medium
- **Time**: 2-3 days
- **Files Changed**: ~8 files
  - New API handler: `legalscanner-api/src/api/handlers/sbom.rs`
  - New model: `legalscanner-api/src/api/models/sbom.rs`
  - Route registration in `routes.rs`
  - UI: Download button + format selector in ResultsViewer.vue
  - Integration tests

#### Technical Details
- New endpoint: `GET /api/v1/scans/:id/sbom?format=json|yaml|tag-value`
- SPDX 2.3 document structure:
  - Package metadata (repository URL, scan timestamp, tool info)
  - File listings with detected licenses and copyrights
  - ECC findings as SPDX annotations
  - Relationships (file ‚Üí package, file ‚Üí license)
- Response: Downloadable file or inline JSON/YAML

### 2. License Compatibility Analysis Engine
**Priority: HIGH ‚≠ê‚≠ê‚≠ê‚≠ê**

#### Why This Matters
- **Catches Compliance Violations**: Mixing GPL + proprietary code = legal nightmare
- **Proactive Risk Detection**: Warns BEFORE shipping incompatible code combinations
- **Educational Value**: Teaches developers about license restrictions and obligations
- **Competitive Advantage**: Most free scanners don't perform compatibility analysis

#### Benefits
‚úÖ **Prevents legal issues** - catches GPL/MIT/Apache conflicts automatically
‚úÖ **Risk quantification** - clear "red flags" for legal review prioritization
‚úÖ **Actionable insights** - tells users what's wrong and why it matters
‚úÖ **Low false positives** - rule-based logic is deterministic and explainable

#### Open Source Library Recommendation
**Recommended: Custom rule-based compatibility matrix**
- **No perfect Rust library exists** (opportunity to create one!)
- Build compatibility matrix manually using authoritative sources
- ~50 common licenses with well-documented compatibility rules
- 2D matrix approach: License A + License B = Compatible/Incompatible/Warning

**Reference Data Sources:**
- FSF License List: https://www.gnu.org/licenses/license-list.html
- OSI Approved Licenses: https://opensource.org/licenses/
- SPDX License List: https://spdx.org/licenses/
- Blue Oak Council rankings: https://blueoakcouncil.org/list
- Google's License Classifier compatibility data

**Alternative Reference: `askalono` (Amazon's license detector)**
- Crate: https://crates.io/crates/askalono
- Pure Rust, good license taxonomy
- Doesn't do compatibility analysis itself
- Could extract compatibility rules from their data/research

#### Implementation Estimate
- **Complexity**: Medium-High
- **Time**: 3-5 days
- **Files Changed**: ~12 files
  - New module: `legalscanner-api/src/analysis/compatibility.rs`
  - Compatibility matrix: `legalscanner-api/src/analysis/compatibility_matrix.rs` (data structure)
  - Database migration: Add `license_conflicts` table
  - API handler: `legalscanner-api/src/api/handlers/compatibility.rs`
  - UI: New "Conflicts" tab in ResultsViewer.vue
  - Documentation for compatibility rules

#### Technical Details
- Algorithm:
  1. Scan all files in repository for licenses
  2. Build license graph (which licenses appear together)
  3. Apply compatibility matrix rules
  4. Detect conflicts (incompatible combinations)
  5. Generate warnings with explanations
- Conflict types:
  - **INCOMPATIBLE**: Cannot be combined legally (e.g., GPL-2.0 + Proprietary)
  - **WARNING**: Requires careful review (e.g., GPL-3.0 + Apache-2.0)
  - **COMPATIBLE**: No known issues (e.g., MIT + Apache-2.0)
- New database table schema:
  ```sql
  CREATE TABLE license_conflicts (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      scan_id TEXT NOT NULL,
      license_a TEXT NOT NULL,
      license_b TEXT NOT NULL,
      conflict_type TEXT NOT NULL,  -- 'incompatible', 'warning', 'compatible'
      explanation TEXT NOT NULL,
      affected_files TEXT,           -- JSON array of file paths
      FOREIGN KEY (scan_id) REFERENCES scans(id) ON DELETE CASCADE
  );
  ```

#### SPDX Document Structure
```json
{
  "spdxVersion": "SPDX-2.3",
  "dataLicense": "CC0-1.0",
  "SPDXID": "SPDXRef-DOCUMENT",
  "name": "Legal Scanner Report - {repo_name}",
  "documentNamespace": "https://legalscanner.example.com/spdx/{scan_id}",
  "creationInfo": {
    "created": "2025-11-01T12:00:00Z",
    "creators": ["Tool: LegalScanner-1.0"],
    "licenseListVersion": "3.21"
  },
  "packages": [{
    "SPDXID": "SPDXRef-Package",
    "name": "{repo_name}",
    "downloadLocation": "{git_url}",
    "filesAnalyzed": true,
    "verificationCode": {
      "verificationCodeValue": "{sha1}"
    }
  }],
  "files": [{
    "SPDXID": "SPDXRef-File-1",
    "fileName": "./src/main.rs",
    "licenseConcluded": "MIT",
    "copyrightText": "Copyright 2025 Example Corp",
    "annotations": [{
      "annotationType": "REVIEW",
      "annotator": "Tool: Semgrep",
      "annotationDate": "2025-11-01T12:00:00Z",
      "comment": "ECC Finding: crypto usage detected (risk: high)"
    }]
  }],
  "relationships": [{
    "spdxElementId": "SPDXRef-Package",
    "relatedSpdxElement": "SPDXRef-File-1",
    "relationshipType": "CONTAINS"
  }]
}
```

#### API Response
- Content-Type: `application/json`, `application/x-yaml`, or `text/plain` (Tag-Value)
